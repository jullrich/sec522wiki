    GraphQL

    Objectives

    Estimated Time: 12 minutes

    In this exercise, we will experiment with GraphQL and learn about the common mistakes of implementing GraphQL based API.

    Requirements

    • [x] Lab VM

    Step by Step

    1. Using Chrome, browse to http://lab-5.2.sans.labs/. You will be presented with a login prompt. Use sec522 / training as username and password to login

    2. Now that you are logged into the application. You see that there are messages posted by multiple users. Let's inspect how this application work. Please toggle the Proxy Switcher to manual proxy mode (it should be red), which means all browser traffic are sent to Burp for inspection.

    3. Switch over to Burp. You should see a request awaiting approval to forward. Under Proxy tab, select Intercept and click forward for the requests. Review the POST request to /graphql, take a look at the request body for the content in JSON format.

    4. In Burp window, under Proxy tab, select the HTTP history which will show you the HTTP requests that were intercepted by Burp. Scroll all the way down and you should find the request and response for the POST request to /graphql. See screen shot for reference.

    5. GraphQL queries are in JSON format and while lightwight, can be a bit difficult to "visualize" the formatting. Luckily Burp has an extension to help navigate around GraphQL. The extension is InQL and requires Jython to run. To get this setup, go to the Extender tab in Burp and select Options sub-tab. Under Python environment, change the location of the Jython standalone JAR file by clicking on the Select File button. The file is located at /home/student/jython-standalone.jar See screenshot for reference

    6. In the tab, select BApp Store to install the InQL extension. There is search box on the upper right hand to help you find the InQL extension in the list. Once you find the InQL - Introspection GraphQL Scanner, click on it and you will get the option to install it at the right side pane (scroll down on the page to see the Install button in orange).

    7. Once the installation if finished, there will be more tabs available (such as InQL Scanner). Go to the Proxy tab to select HTTP History, we are once again looking for the POST request to the /graphql (scroll down on the list). Once you find the request in the history, right click on it. In the context menu, select Copy URL.

    8. Change over to the InQL Scanner tab. Paste the URL at the top of the window and then click on Load at the right hand side. A request is generated that you will need to click Forward under the Proxy -> Intercept tab. Once done, flip back to InQL Scanner. Some data are appearing at the pane to the bottom. InQL is using introspection queries to determine all the queries supported at this GraphQL endpoint. This should help you answer the first challenge in NetWars.

    For second NetWar challenge, you will need to generate multiple queries to the GraphQL endpoint.

    1. In Chrome, browse to http://lab-5.2.sans.labs/settings Since Burp is in interception mode, you will need to change to Burp to click Forward (multiple times) under the Proxy -> Intercept tab.

    2. In Burp window, under Proxy tab, select the HTTP history which will show you the HTTP requests that were intercepted by Burp. Scroll all the way down and you should find the request and response for the POST request to /graphql. Right click on the request in history and then in the context menu, select Send to Repeater.

    3. Select the Repeater tab. You should be presented with the POST /graphql request ready to be sent. As discussed earlier, the JSON data can be difficult to visualize and manipulate. InQL extension made this a lot easier. Click on Select Extension and select InQL. Refer to the screenshot below

    4. At this point, the pane should be just the GraphQL query for edit and send. This should help with the NetWars challenge. Feel free to take hints in NetWars if necessary.

    Free hint

    • Pay attention to user: 1

    Conclusion

    • GraphQL offers single endpoint data manipulation and querying capability.
    • Introspection is powerful and allow developers to understand the remote endpoint's capability. This is not meant to be for public production access as it yield valuable information to the attackers.
    • IDOR can be a significant challenge with GraphQL based API.
