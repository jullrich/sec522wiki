    Exercise 2.1 - SQL Injection

    Objectives

    Estimated Time: 15 minutes
    Part 1:

    You will familiarize yourself with the basic principles of SQL injection.

    Part 2:

    You will experiment with a less common SQL injection to learn how SQL injection may be exploited to do more than retrieve data.

    Requirements

    • [x] Lab VM

    Part 1: Basic SQL Injection

    No Hints

    We are again using our "Animal Shelter" application. This time, we use a modified page that includes a search feature. The search feature allows for SQL injection. You can check the "Debug" checkbox to see the SQL query. The page can be found at http://lab-2.1.sans.labs/.

    Step by Step

    1. Open http://lab-2.1.sans.labs/ in Google Chrome.

    2. Enter one of the animal names (for example, "Stash") in the search box and click "Submit." Only this animal should now be visible.

    3. Enter a single quote ( ' ) in the search box and click "Submit." This should trigger an SQL error. The SQL error message will include the query. Try to identify the vulnerability.

    Digging Deeper

    Critical: The next few commands require you to include a space after the final '--'.

    1. Enter the string below in the search field and click submit. You will not see an error now, but all animals are visible.

      ' OR 1=1 --

    2. Enter the below string and click "Submit."

      ' UNION SELECT 1,2,3 --

      The last step will again result in an error message. The error message will state that both SELECT statements in a UNION query need to return the same number of columns. Count the columns returned by the first statement and rewrite the second statement using the correct number of columns.

    3. Enter the below string and hit submit. You will now get a broken image, and the numbers 2 and 4 will be visible below the broken image.

      ' UNION SELECT 1,2,3,4,5,6,7 --

      Result of Entering the Correct UNION Query

    4. Replace the SELECT query with one selecting the username and password column from the "animalshelter.users" table

      ' UNION SELECT 1,username,3,password,5,6,7 FROM animalshelter.users --

      Do not forget the space after the -- in the end.

      You will now see two broken images (one for each user in the database). The usernames and passwords are displayed where you saw the numbers 2 and 4 in Step 6.

    5. View the page source (CTRL-U) to see the password hashes more clearly.

      You may try other queries now.

    Can you create a query without the '--' in the end?

    Answer: ' OR 1='1

    The challenge in NetWars pertains to this input form. If necessary, NetWars can offer hints at the cost of some points.

    Part 2: Advanced SQL Injection

    No Hints

    The login page at http://lab-2.1.sans.labs/login.php is vulnerable to SQL injection. Use the vulnerability to log in as an administrator.

    Step by Step

    We have a URL at hand that we know is vulnerable to SQL injection. The first step when exploring SQL injection is to insert a quote into each field.

    1. Use Chrome to visit http://lab-2.1.sans.labs/login.php.

    2. Login using the username "sec522" and the password "training." (to assist in exploiting the SQL injection vulnerability, passwords are not masked on this page)

    3. You are now logged in. Click on the "Log Out" link to return to the login page.

    4. Enter a single quote as the user name. You will get a SQL error message that displays the SQL query. You can inject SQL via the username. You may use a UNION query to trick this query into returning whatever password you entered, not the actual password for the user admin. Let's try this by connecting to the database directly.

    5. Open an OS terminal window.

    6. Enter the command,

      mysql animalshelter

      to connect to the "animalshelter" database.

    7. Enter the query,

      select username, password from users where username='admin';

      You will receive a response like

      +----------+--------------------------------------------+
| username | password                                   |
+----------+--------------------------------------------+
| admin    | $2y$10$/Fsy3WqrnbEKT2IisqgTEdwygLz.SYOnLMW |
+----------+--------------------------------------------+

    8. We do not know the administrator's password. We need to rewrite the query to return whatever password we entered. Use the tool at http://lab-2.1.sans.labs/pwhash.php to calculate the password hash for the password of your choice (keep it simple).

    9. We have to create a UNION query to return the password hash we need. You may experiment with the MySQL command line. For example, for the password abc, the query would look like this:

      SELECT username, password FROM users WHERE username='' UNION SELECT 'admin','$2y$10$G0aDOmI7vQsmfUw2[part removed]Wzy6S'

    10. Now create the correct username to log in as admin with the password abc (or whatever password you selected). The flag for the scoreboard will be displayed once you are successful.

    Conclusion

    • SQL injection may impact much more than just the data within a single request. Passwords and personal information leakage are commonly the results of SQL injection. But it may extend to the compromise of an entire network.
    • SQL queries often govern business logic and access control. SQL injection can be used to bypass these rules.
